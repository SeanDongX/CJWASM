## 剩余问题分析

### 问题 1: 接口方法调用
**根本原因**: 接口是抽象的，其方法没有具体实现。接口方法调用需要通过实现类的虚函数表。

**可能的解决方案**:
1. 不编译接口的默认方法为独立函数
2. 将接口默认方法内联到实现类中
3. 实现完整的接口虚函数表机制

**建议**: 暂时跳过包含接口默认方法的模块（crypto, io）

### 问题 2: 类型名作为值
**根本原因**: 某些代码中使用类型名作为值（可能是反射或泛型相关）

**需要调查**:
- 找到具体使用 `Int64` 作为值的代码位置
- 理解使用场景
- 决定是否支持类型字面量

**建议**: 使用 RUST_BACKTRACE 找到具体文件和行号

### 问题 3: HashMap 未定义
**根本原因**: unicode 模块使用了泛型 HashMap，但没有被单态化

**可能的解决方案**:
1. 确保 collection 模块在 unicode 之前加载
2. 检查泛型单态化逻辑
3. 手动注册常用的 HashMap 实例
